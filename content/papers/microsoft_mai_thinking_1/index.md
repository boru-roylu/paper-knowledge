## 一句話總結
**MAI-Thinking-1** 是 Microsoft AI 從零訓練的 reasoning model technical report；重點不是單一模型架構新招，而是把 data pipeline、pre-training ablation、RL recipe、evaluation、safety、training infrastructure 串成一台可持續改進模型的 **hill-climbing machine**。

## 這篇在解決什麼問題
這篇的問題意識是：frontier model progress 不是靠一次性訓練一個模型，而是靠能反覆、可靠、可量測地改進模型的系統。作者主張模型開發應該被視為 system-level optimization problem，核心是建立能長時間 hill-climb 的工程與研究流程。

它反對兩個常見捷徑：

- **只靠 distillation**：能快速得到能力，但作者認為會犧牲 steerability、robustness，以及長期 RL climb 的可控性。
- **只看單點 benchmark**：模型決策必須透過 scaling ladders、ablation、held-out NLL evals、RL curves、safety evals 一起驗證。

MAI-Thinking-1 是這套流程的第一個產物：一個 **35B active / 1T total parameter sparse MoE** reasoning model。

## 核心方法
### 1. Pre-training architecture
MAI-Base-1 是 decoder-only Transformer，核心設計包括：

- 35B active / 1T total sparse MoE。
- periodic local/global attention：5 個 local attention layers 搭配 1 個 global attention layer。
- local attention 使用 RoPE 和 sliding window 512；global attention 不使用 position encoding。
- group-query attention，8 KV heads，head dimension 128。
- RMSNorm、SwiGLU、tied input/output embeddings。
- tokenizer 使用 `o200k_base`，vocab size 200,019。
- feed-forward blocks 採 dense FFN 和 sparse MoE 交錯。
- MoE 使用 LatentMoE design；每 token 啟用 **8 / 512 experts**，在 compressed latent space dispatch。

作者的重點是 architecture 與 infrastructure co-design：不是只追求理論 FLOPs，而是要在 GB200 cluster、training throughput、RL serving、checkpoint/restart 上都可持續運作。

### 2. Data pipeline / pre-training
Pre-training 使用 **30T tokens**，之後 mid-training 再加 **3.55T tokens**。資料來自 publicly available 和 licensed human-generated data，包括 web、public GitHub code、books、academic papers、news、multilingual text、domain-specific material。

關鍵資料政策：

- 不使用第三方模型 distillation data。
- pre-training 不使用 synthetic data generated by language models。
- 盡力避免和移除 AI-generated content。
- 不使用 open-source training datasets。
- 對 common ML databases 做 decontamination。

資料管線重點在 dedup/filter/categorization：

- boilerplate removal
- exact duplicate removal
- MinHash LSH fuzzy deduplication
- templated web page skeletonization
- semantic duplication clustering
- cross-dataset deduplication with global drop-order
- spam / restricted / policy-sensitive content filtering

### 3. Reinforcement learning climb
Pre-/mid-training 只提供 broad predictive competence；RL climb 讓模型學會 reasoning、tool use、human preference、safety behavior。

作者描述的 RL climb 特點：

- 從零開始學 reasoning，沒有先看 reasoning traces。
- 使用 task-specific feedback 讓模型學 CoT。
- 訓練三個 domain specialist models：STEM reasoning、agentic coding/tool use、helpfulness & safety。
- 透過 robust RL recipe、self-distillation、infrastructure improvements，維持 thousands of steps 的 log-linear climb。
- 最後把能力 consolidation 到 single model。

### 4. Infrastructure
Technical report 很大一部分在講 training stack。幾個值得記的點：

- pre-training 在 Microsoft-operated Azure cluster 上使用 **8K GB200 GPUs**。
- YOLO distributed training stack 追蹤 MFU 和 useful-FLOP throughput。
- checkpointing 改寫 PyTorch DCP metadata / serialization path，使 CPU memory overhead 和 checkpoint save time 降低超過 10x。
- 使用 asynchronous checkpointing、custom coordination layer、Azure Blob Storage local deployment、Ray actor hot standbys、restart determinism validation。
- training loss 在早期遇到 spikes，但沒有跳 batch，也沒有中途手動調 training config。

## Training / Data
這篇沒有開放完整資料集，但披露了 high-level recipe：

- main pre-training：30T tokens。
- mid-training：3.55T tokens，強化 STEM、math、coding。
- max context length：mid-training 後達到 **256K**。
- no third-party distillation / no synthetic pretraining data。
- train data source 全部 in-house processing。
- RL 使用 STEM、agentic coding、helpfulness/safety 等 task feedback。

對 agentic coding，paper 特別說 SWE agentic training data 只使用 bash 和 string-replace tools，沒有 targeted Terminal-Bench-like environments；因此 Terminal-Bench 成績更像 broader agentic training 的 generalization。

## 主要結果
MAI-Thinking-1 在 public benchmarks 上屬於強 reasoning / coding model，但 paper 也明確說它不是全面領先 frontier。

關鍵數字：

- AIME 2025：**97.0**
- AIME 2026：**94.5**
- HMMT Feb 2026：**84.9**
- GPQA Diamond：**84.2**
- LiveCodeBench v6：**87.7**
- Terminal-Bench 2.0：**46.0**
- SWE-bench Verified：**73.5**
- SWE-Bench Pro：**52.8**

與 Claude Sonnet 4.6 的 broader comparison：

- MMLU Pro：85 vs Sonnet 87
- SimpleQA Verified：31 vs Sonnet 29
- IFBench：69 vs Sonnet 50
- Advanced IF：85 vs Sonnet 86
- Multi-Challenge：53 vs Sonnet 57
- GraphWalks <=128k：90 vs Sonnet 96
- BFCL v3：72 vs Sonnet 76

Safety / honesty / health 類：

- AIR-Bench：88，與 Sonnet 4.6 相同。
- CyberSec Instruct / Auto：63 / 63，略高於 Sonnet 62 / 56。
- Long Fact：98，與 Sonnet 98 相同。
- TruthfulQA：88，與 Sonnet 88 相同。
- HealthBench Prof. / MedXpert QA：35 / 43，低於 Sonnet 38 / 49。
